# Тесты

Автотесты фронта магазина. Писала на TypeScript: unit и integration на Vitest, e2e на Playwright.

Unit — без браузера и сети.  
Integration — уже с fetch/localStorage, но зависимости мокаются.  
E2E — как пользователь: открыла страницу, кликнула, проверила результат.

## Как устроено

Структура папок такая же, как в оригинальном проекте — просто вынесла сюда тестовую часть:

tests/
setup/ — общий setup для vitest
unit/ — мелкие чистые функции
integration/ — api + localStorage
e2e/
helpers/ — куки / попап города
pages/ — page objects
specs/ — сценарии в браузере

## Что внутри

Unit: склонение отзывов, формат цены, валидация количества, хлебные крошки каталога.

Integration: логин по телефону и смс-код, фильтр товаров через api, корзина и избранное в localStorage.

E2E: главная, каталог, поиск в шапке, переход в корзину. Для e2e использовала Page Object, чтобы не дублировать селекторы.

Тут только малая часть из того, что покрыто в основном — вынесла самые показательные кейсы.

## CI

В GitLab тесты гоняются перед деплоем: сначала Vitest, потом отдельно Playwright. Деплой на main не идёт, пока unit/integration красные. E2e специально с allow_failure: они ходят на внешний api, и если он временно недоступен или тормозит, пайплайн из‑за этого не считается упавшим — unit/integration при этом всё равно обязательные.
